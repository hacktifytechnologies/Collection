# T1074.001 – Local Data Staging | Assessment

## MCQ 1
Attackers use `/dev/shm` for staging because:
A) It has faster read speeds  B) It is a RAM-backed tmpfs — files exist only in memory and disappear on reboot, leaving minimal forensic artifacts  ✅
C) /dev/shm is never monitored  D) It bypasses file system permissions

## MCQ 2
Staging data before exfiltration serves to:
A) Encrypt the data permanently  B) Consolidate multiple collected files into a single organized archive, making exfiltration more efficient  ✅
C) Compress the data for permanent storage  D) Make data readable by the C2 server

## MCQ 3
A forensic investigator would look for staging artifacts in:
A) Only /tmp  B) /tmp, /dev/shm, /var/tmp, hidden dotfiles, and recently created archives  ✅
C) Only /root  D) Only network share mounts

## Fill 1
`tar` flag combination to create a compressed archive from a directory:
**Answer:** `czf` (or `czf archive.tar.gz -C /source .`)

## Fill 2
Linux RAM-backed filesystem directory that loses all contents on system reboot:
**Answer:** `/dev/shm`
