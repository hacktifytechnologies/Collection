# T1074.001 – Local Data Staging | Solution Walkthrough
**Difficulty:** Easy | **OS:** Linux | **MITRE:** T1074.001

## Step 1 – Stage Files to Hidden Directory
```bash
# Copy all loot to staging area:
cp -r /opt/loot-source/* /tmp/.stage_area/
ls -la /tmp/.stage_area/
# configs/  creds/  docs/
```

## Step 2 – Compress Staged Data
```bash
tar czf /tmp/.stage_area/loot.tar.gz -C /opt/loot-source .
ls -lh /tmp/.stage_area/loot.tar.gz
# Inspect contents:
tar tzf /tmp/.stage_area/loot.tar.gz
```

## Step 3 – Encrypt Before Exfil
```bash
openssl enc -aes-256-cbc -k 'StageKey' -pbkdf2 \
    -in /tmp/.stage_area/loot.tar.gz \
    -out /tmp/.stage_area/loot.enc
# Now ready for exfiltration — encrypted + compressed
```

## Step 4 – Find Flag
```bash
grep HACKTIFY /opt/loot-source/configs/main.conf
# HACKTIFY{LOCAL_DATA_STAGING_<hash>}
```

## Common Staging Locations
```bash
/tmp/.hidden_dir/       # Hidden in /tmp
/dev/shm/               # RAM-based filesystem (disappears on reboot)
/var/tmp/               # Persists across reboots
C:\Windows\Temp\        # Windows equivalent
C:\ProgramData\         # Windows — usually not inspected
```

## Anti-Forensics During Staging
```bash
# Use RAM filesystem (no disk writes):
mkdir -p /dev/shm/.loot && cp -r /opt/loot-source/* /dev/shm/.loot/
# Files in /dev/shm vanish on reboot — minimal forensic footprint
```
